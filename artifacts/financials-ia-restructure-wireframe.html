Financials section — information-architecture restructure

This is a picture of a proposed navigation layout, not a built feature. No app code, routes, or roles have been changed. The goal: give bank-linking a correctly-named home and group the eight Financials items so the section reads cleanly. Wireframe fidelity (boxes + labels) on purpose.

The naming collision we're fixing

Two different things are both trying to be called "Payment Methods."
• The Plaid bank-linking page (/app/financial/bank-connections) shows a sidebar label of "Bank Connections" but its own page header reads "Payment Methods" — a self-contradiction that confused even the team.
• Meanwhile the real Payments page (/app/financial/payments) is the genuine "how owners pay their dues" surface (payment channels + Stripe Connect). That is where "Payment Methods" actually belongs.
Fix: bank-linking becomes "Bank Accounts" (connecting the association's own bank accounts via Plaid for reconciliation). Owner-facing "Payment Methods" lives inside Payments, where it already is.

Proposed grouping — every move, one-line reason

GroupItem (label)What the page doesWhy it lives here
Setup Chart of Accounts
board · manager · admin
 Define the account tree + budget categories the whole section posts against. Configured once at the start; everything else depends on it.
Setup Bank Accounts was "Bank Connections" / header "Payment Methods"
board · manager · admin
 Connect the association's own bank accounts via Plaid so transactions sync for reconciliation. Connecting the HOA's banks is a one-time setup act — not a way owners pay. Renaming kills the "Payment Methods" collision.
Setup Assessment Rules
board · manager · admin · viewer
 Rules that auto-generate recurring assessments / charges. You set the rules up front; billing then runs them. Belongs with the configuration items.
Operations Billing
board · manager · admin · viewer
 Generate and track the dues/assessments charged to owner ledgers. The core recurring money-out-the-door activity.
Operations Payments keeps "Payment Methods"
board · manager · admin
 How owners pay dues — payment channels owners see, plus the Stripe Connect gateway. This is the real owner-facing "Payment Methods." Keeping the name here resolves the conflict cleanly.
Operations Expenses
board · manager · admin
 Record and track money the association spends (invoices, utilities, budgets). The money-out counterpart to Billing; a monthly operational task.
Insight Reports
board · manager · admin · viewer
 P&L, Collections, AR Aging, Reserve, Board Report. Read-only output that reads from everything above — naturally last.
Insight Owner Statement was "Statement"
board · manager · admin · viewer
 A single owner's account statement (charges, payments, balance). Also read-only reporting; "Owner Statement" says what it is at a glance.
