[PROJECT CHARTER]

Mission. Take YourCondoManager from its current state to "any HOA/COA, confidently and smoothly managed, in any of the 50 states" — a financial-core-first platform driven by a configurable compliance rules engine, at feature parity with incumbents (PayHOA / CINC / Vantaca / Buildium / FRONTSTEPS) and winning on the trust/money/risk white space, expanded one state-cluster at a time.

Scope IN.

  • The financial-operations core (fund-accounting GL, bank reconciliation, dues collection, AP, 1099) and its external rails.
  • The configurable compliance rules engine (per-state requirements as data, not code) — the architectural keystone.
  • State-cluster expansion in sequence: B (baseline) → A (UCIOA reserve) → C (structural-integrity) → D (Florida + California).
  • White-space differentiators layered on parity (fraud/dual-control first).
  • The AUDIT-HOA-001 gap audit that baselines current-vs-target and feeds the roadmap.

Scope OUT (explicit — routed elsewhere).

  • The marketing website + brand redesign — that's the separate redesign effort. → YCM redesign track.
  • Day-to-day live-tenant ops (CHC dues, the in-flight Plaid production sign-up, support). → YCM GM / CoS.
  • Asserting specific state-law rules as settled fact. → counsel owns the legal verification; the platform carries a "verify with counsel" disclaimer.
  • Pricing / GTM / banking revenue-share strategy. → founder + GM decision, not this build track.

Success criteria.

  • AUDIT-HOA-001 baseline complete with an honest current-state scorecard.

  • The make-or-break financial trio — F1 fund-accounting GL, F2 bank reconciliation, F3 dues collection — reach Working (reconcile >95% of transactions; dues end-to-end).
  • Cluster B serveable, then Cluster A.
  • ≥1 white-space differentiator shipped (fraud dual-control safeguards).

Deliverable shape. This versioned project artifact + the AUDIT-HOA-001-results.md baseline + a cluster-gated build roadmap that feeds dispatches.

Owner-routing map. Legal specifics → counsel · brand/site → redesign track · live ops → YCM GM/CoS · pricing → founder.

Target Architecture — the destination

Every leading platform is built around the same spine: a community-association-aware fund-accounting general ledger (operating vs. reserve money separated) wired into external rails. Money never flows through the vendor's own accounts — Stripe/Plaid route bank-to-bank (the correct compliance posture).

Must-have integration stack (synthesis)

Plaid + a community-association bank direct API (Alliance Association Bank / Pacific Premier / CIT) + Stripe + a lockbox + AvidXchange-class AP + Twilio + SendGrid + Lob + DocuSign + an e-voting provider + a reserve-study data feed — all on a fund-accounting GL hosted SOC 2 / PCI-compliant, multi-region.

Make-or-break trio: fund-accounting GL (F1), bank reconciliation (F2), dues collection (F3). Without these Working, the platform is pre-MVP no matter how many other features exist.

50-State Compliance Model

The keystone decision: per-state requirements must be configuration, not code. Adding a state = data entry + legal review, not an engineering change. If state logic is hard-coded, the platform is capped at one cluster regardless of feature completeness — so the rules engine is the highest-leverage architectural bet.

White-Space Edge — where to beat incumbents

The frontier moved from "software that records work" to "agentic AI that does the work" (Vantaca HOAi, CINC Cephai, AppFolio Realm-X). An AI assistant is now table stakes, not a differentiator. The real white space is in trust, money, and risk:

RankOpportunity (pain × scarcity)Status in market
1Fraud / financial controls — real-time anomaly detection, ghost-vendor/duplicate-invoice flagging, enforced dual-control disbursement, Positive Pay/ACH-blockEndemic problem ($58M Las Vegas scheme; $11M Hammocks). Essentially no software-native solution. → market as "embezzlement-proof"
2Insurance-crisis tooling — coverage-vs-replacement-cost, gap/non-renewal alerts, renewal packaging, deductible-shift modeling89% of HOAs saw premium hikes in 2024; near-empty in software
3Structural / reserve transparency (post-Surfside) — SIRS/milestone deadline tracking, reserve-health vs. baseline, auto resale disclosureRegulatory tailwind; thin coverage
4Sentiment / at-risk-resident detection + ADR/dispute toolingConfirmed empty in HOA software
5Landlord-owner & STR compliance — lease/tenant tracking, rental-cap enforcement, embedded Airbnb monitoringOnly standalone govtech tools; embed opportunity
6–8Cross-network benchmarking · embedded fintech (insured reserve sweeps + controls) · open hardware API ecosystemPlatform moats (need data scale)

Lead with #1. Dual-control approval workflows + duplicate/ghost-vendor flagging are low-effort, high board-trust payoff, and directly attack the biggest gap — a strong Stage-1 differentiator that pairs with the financial core.

Gap Audit — AUDIT-HOA-001 (the first move)

Before any roadmap is real, baseline the current codebase against the target. The audit is evidence-based (file path / route / migration / Issue — never marketing copy) and reuse-first (a capability that exists undiscovered is a documentation gap, not a feature gap).

Three-state capability model

StateBar
DesignedSpecced/stubbed — intent exists, doesn't run
LiveRuns in some environment, happy-path only, not proven end-to-end
WorkingDoes the real job end-to-end with real data, incl. unhappy paths — an HOA could rely on it
AbsentNone of the above (legitimate, common — most compliance layer expected Absent)

What the audit must verdict

  • E1 — is there a configurable rules engine? (Adding a state = data, not code?) The single most important finding; it caps the ceiling.
  • F1 / F2 / F3 — fund-accounting GL, bank reconciliation, dues collection: trace a real transaction end-to-end; rate strictly. Confirm money never touches platform accounts.
  • Rails, not just features — none / stub / sandbox / one-real / all-wired for each integration. A feature can't be Working on a stubbed rail.
  • Cluster readiness — which of B/A/C/D can we serve today, and what specific feature unlocks the next.

Output

A report at audits/AUDIT-HOA-001-results.md: headline findings → capability scorecard → rails readiness → cluster verdict → ranked gap list (worst-impact first) → smallest set of next builds, each tied to a benchmark. Plus appendices for unmapped/extra capabilities and documentation gaps.

Recommended first action: run AUDIT-HOA-001 against the current YourCondoManager codebase. Everything downstream (roadmap sequencing, cluster claims) depends on this honest baseline.
