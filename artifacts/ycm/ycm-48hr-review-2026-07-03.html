What we shipped

All live, all verified. Ten things landed in 48 hours.

LiveReal financial ledger for Cherry Hill

Cherry Hill now runs on a proper double-entry general ledger with real financial statements — the accounting backbone a managed association needs. It's built to show any out-of-balance amount right in a banner, so a problem is surfaced, never hidden.

LiveThe catch that matters — books were overstating owner debt by ~$12,500

The moment the new reconciliation ran, it caught it: 70 owner payments had landed in the Chase bank account but were never credited to the owners, so the books showed people owing money they'd already paid.

Root cause: 17 of Cherry Hill's 19 owners had no "association" link, so the matching engine could only see 2 of them and matched nothing. Fixed — the links are set, the engine now sees all 19.

LiveOne-click payment resolution screen

Applying those 70 payments is now a click-through, not a data project. Each Chase deposit shows a suggested owner and a confidence level, so you approve the clear ones fast and only eyeball the uncertain few.

LiveOwner portal fix — balances read honestly

Paid-up units now show "Paid in full on <date>", and a large balance sitting on a payment plan no longer looks alarming. Owners see an accurate, calm picture of where they stand.

LiveData-integrity sweep — the same bug found & fixed elsewhere

We swept the whole platform for the same "no association link" bug and found + fixed it in 3 other communities (6 more owners). We also set primary contacts on 15 Cherry Hill units — that went from 1 of 18 to 17 of 18.

DoneTwo owner name / title corrections

Applied to two units — both zero-balance, so no money was affected.

LiveCash Flow report added — reporting is now complete

The financial reporting set is now the full suite: P&L, budget-vs-actual, AR aging, delinquency, cash flow, and a board summary. Nothing missing for a board meeting.

DoneReusable new-client onboarding template

The entire Cherry Hill setup was turned into a repeatable playbook plus a data-readiness checklist — so onboarding the next client is a known path, not a from-scratch build.

LiveStripe payments webhook fixed this morning

The live payments webhook was erroring — Stripe would have shut it off on July 9. Root cause: it was being sent billing events it wasn't built to handle. Fixed to acknowledge them correctly. Payments keep flowing.
