Check 1 — Owners with a blank association that own a unit AUTO-FIXED · reversible

This is the exact bug class from the 17-owner Cherry Hill incident: a person record with no association_id that nonetheless owns a unit — which breaks reconciliation because the owner isn't scoped to a community.

Found 6 such owners across 3 communities. Every one owned units in exactly one association (none ambiguous), so each was safely repaired: association_id set to that community, inside a transaction, with a full before/after audit_logs row (action=update.person.association_id). 0 skipped.
