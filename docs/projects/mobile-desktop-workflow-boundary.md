# Mobile Desktop Workflow Boundary

This document records which board workflows are considered mobile-safe for review and which still require desktop-first authoring.

## Mobile Review Supported
- Board roster review: role, association, term state, and expiry status
- Board package review: template scan, generated package review, annotation review, distribution history
- Meeting review: meeting register, quorum check, meeting notice send, note access, summary publish toggle
- Resolution review: resolution selection, vote recording, vote totals
- Governance compliance review: template item scan, task due-state review, status updates, evidence links/uploads
- Board activity and board-facing operational summaries in the owner portal

## Desktop-Preferred Authoring
- Long-form packet authoring or heavy restructuring of package content
- Drafting and editing large meeting minutes or highly formatted long-form summaries
- Deep template composition where many sections, references, or source records are edited at once
- Regulatory or compliance bulk editing that depends on scanning many fields simultaneously

## Handoff Messaging Rule
- If a desktop-preferred authoring flow is reached on mobile, show an explicit message that the workflow is best completed on desktop.
- Do not label those authoring paths as mobile-ready until the editing surface is genuinely usable at the required viewport matrix.

## Current Explicit Desktop-Preferred Candidates
- Board package content authoring with dense multi-section editing
- Long-form draft minutes editing and final formatting
- Large-scale compliance template maintenance or source-library editing
