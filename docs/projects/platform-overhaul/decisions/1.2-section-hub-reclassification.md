## Conflict Analysis

Seven decision axes from the inventory: universality, composition, low-surface-area sections, persona adaptation, per-section classification, hub-vs-redirect coexistence, and naming convention. Each is resolved below.

## Selected Resolutions

### Q1 — What is the definition of a "section hub"?

**Resolution:** A section hub is the top-level landing page for a zone. It is reached by clicking a zone label in the sidebar. It is a navigational and summary surface — not a dashboard, not a workflow page. Its primary job is to orient the user within the zone and provide direct access to sub-pages.

**Acceptance Criteria**
- Every zone has exactly one hub page
- Hub pages are not workflow pages — they contain no forms and no deep data tables
- Hub pages are reachable directly from the sidebar zone label
- Hub pages are not the same as the Home dashboard

### Q2 — What does a hub page contain?

**Resolution:** Every hub contains: (1) zone title matching the sidebar label, (2) one-sentence zone description in plain language, (3) a list of direct links to primary sub-pages within that zone, (4) optionally up to three quick-action shortcuts for the highest-frequency actions in that zone, (5) optionally one lightweight summary widget where it adds genuine value (e.g., outstanding balance in Financials, open requests in Operations). Hubs do not contain more than one summary widget, duplicate content from Home, deep tables, or full workflow forms.

**Acceptance Criteria**
- Zone title on hub matches sidebar label exactly
- Zone description is one sentence, plain language
- Sub-page link list is present and complete for that zone
- Quick-action shortcuts, if present, do not exceed three
- Summary widget, if present, does not exceed one per hub
- No hub contains a data table or a workflow form

### Q3 — Which of the 13 existing sections become hubs, and which become sub-pages?

**Resolution:** VS Code agent must map all 13 sections to the five-zone model and designate one page per zone as the hub. The hub is the most general/overview page in that zone. All other pages in the zone are sub-pages. The mapping table is a required implementation output — it is not pre-assigned here because the 13-section inventory has not been fully audited against the new zone model. Any section that does not map cleanly to a zone is escalated before assignment.

**Acceptance Criteria**
- All 13 sections are mapped to one of the five canonical zones
- Each zone has exactly one designated hub page
- The mapping table is committed to `docs/projects/platform-overhaul/decisions/1.2-section-hub.md`
- No section is silently dropped — unmapped sections are flagged as open items

### Q4 — What is the hub URL convention?

**Resolution:** Hubs live at the zone root URL:
- `/app/financials` → Financials hub
- `/app/operations` → Operations hub
- `/app/governance` → Governance hub
- `/app/communications` → Communications hub
- `/app` → Home (canonical dashboard, not a hub)
- `/portal/finances` → My Finances hub
- `/portal/requests` → My Requests hub
- `/portal/community` → My Community hub
- `/portal` → Owner Home (canonical dashboard, not a hub)

**Acceptance Criteria**
- Each zone hub is accessible at its zone root URL
- No hub lives at a sub-path of the zone root
- Home and Owner Home are dashboards, not hubs, and are exempt from hub content rules

### Q5 — Does every zone get a hub, including Home?

**Resolution:** All zones except Home get a hub. Home at `/app` and `/portal` is the canonical dashboard — it has different content rules and is not a navigational hub. Home is the landing page for the workspace, not a zone entry point.

**Acceptance Criteria**
- Four hubs exist in `/app`: Financials, Operations, Governance, Communications
- Three hubs exist in `/portal`: My Finances, My Requests, My Community
- `/app` and `/portal` are dashboards, not hubs, and do not follow hub content rules

### Q6 — Are hubs role-filtered?

**Resolution:** Yes. Hub pages display only the sub-page links and content relevant to the active persona. A Board Admin seeing the Financials hub sees a different sub-page list than a Manager. The hub component renders based on the authenticated user's role.

**Acceptance Criteria**
- Hub sub-page lists are filtered by persona
- A user never sees a link on a hub to a page they cannot access
- Hub filtering uses the same role-gating model as the sidebar

### Q7 — What is the naming convention for hub pages — "Overview," "Hub," or section name?

**Resolution:** Hub pages use the **zone label alone** as their display title. No suffix. The Financials hub is titled "Financials." The Operations hub is titled "Operations." The word "Hub" never appears in a page title. "Overview" is reserved for dashboard-type pages (e.g., Association Overview, Portfolio Overview) and must not be used as a hub title.

**Acceptance Criteria**
- Hub page `<h1>` matches the zone label exactly
- No hub page title includes the word "Hub" or "Overview"
- Document title format follows 1.4: `Financials — YCM`, `Operations — YCM`, etc.
