# 5.4 — Performance Audit

**Status:** SPEC LOCKED (Wave 14, 2026-04-24) · **IN BUILD (partial)**
**Owner:** Platform Overhaul
**Layer:** 5 — Polish & Hardening

---

## Scope

Baseline perf audit across three axes:
1. **Bundle** — measure `dist/` chunk sizes after `npm run build`. Identify the biggest imports if any client chunk exceeds 500 KB gzipped, and document (do not refactor in Wave 14).
2. **Server endpoints** — survey the top ~10 traffic endpoints for query complexity (N associations × M rules, etc.). Document worst-case Big-O and DB round-trips.
3. **Image optimization** — spot-check static assets for unoptimized images.

Wave 14 is a **scoping doc + baseline**, not a refactor wave. All follow-up work is filed at the bottom of the audit doc.

Also adds a minimal smoke perf test: time one endpoint invocation against mocked DB and assert completion under 500ms. This documents a baseline — it is not a real load-test.

---

## Acceptance Criteria

1. `implementation-artifacts/5.4-performance-audit.md` documents:
   - Bundle output sizes (main + largest 3 chunks, both raw and gzipped when available).
   - Endpoint complexity notes for: `/api/alerts/cross-association`, `/api/operations/dashboard`, `/api/portal/financial-dashboard`, `/api/financial/assessment-run-log`, `/api/financial/rules/:ruleId/run`, `/api/portal/me`, `/api/admin/billing/subscription`, plus 3 more picks.
   - Top 3 recommendations for future work.
2. Wave 14 does NOT refactor server code to address performance — each identified item is filed as a follow-up line in the audit doc.
3. A smoke perf test (`tests/perf-smoke.server.test.ts` or similar) invokes one endpoint path against mocked dependencies and asserts completion under 500ms. The test is a baseline, not a regression gate.
4. No new external dependencies (no bundle analyzer GUI, no APM).

## Bundle Budget (Wave 14)

- **Tier A (fail audit):** any single client chunk > 1 MB gzipped.
- **Tier B (file follow-up):** any single client chunk > 500 KB gzipped.
- **Tier C (note only):** largest chunk between 250–500 KB gzipped.

Wave 14 records the tier and files the follow-up — it does not block the PR on Tier A/B unless explicitly called out.
