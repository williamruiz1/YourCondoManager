# 5.1 — Empty States

**Status:** SPEC LOCKED (Wave 14, 2026-04-24) · **IN BUILD (partial)**
**Owner:** Platform Overhaul
**Layer:** 5 — Polish & Hardening

---

## Scope

Consistent empty-state UX across every list/table in the operator workspace (`/app/*`) and owner portal (`/portal/*`).

Today each surface rolls its own empty-state copy and markup — inline `<p>`, centered `<Card>`, or nothing at all. This drives inconsistent tone, inconsistent CTAs, and makes it easy to ship a new list with no empty state at all.

This module introduces a shared `<EmptyState>` component and adopts it on a triage-ordered set of surfaces. Full retrofit across the ~60 list/table surfaces is out of scope for Wave 14 — remaining surfaces are tracked in `implementation-artifacts/5.1-empty-states-adoption.md`.

**In scope:**
- Shared `client/src/components/empty-state.tsx` component.
- Adoption on the explicitly-called-out high-traffic surfaces (below).
- Tests asserting the empty state renders when the list is empty.

**Out of scope:**
- Rewriting surfaces that already have a bespoke empty state that's working (Home alerts panel "all clear", HubAlertWidget "all clear", central inbox per-filter empty states — these are already shipped and aligned).
- Animated / illustrated empty states. Stick to icon + title + description + optional CTA.
- Portal list surfaces that already render a reasonable empty state and have no CTA need.

---

## Acceptance Criteria

1. `client/src/components/empty-state.tsx` exists and exports `EmptyState` with props `{ icon, title, description, cta?: { label, onClick | href } }`.
2. Icon is a `LucideIcon` component (passed as a reference, not a rendered element) — the component renders it with consistent sizing (40px) and muted color.
3. `cta.onClick` renders a `<Button>`; `cta.href` renders a `<Link>` styled as a button. If both are present, `onClick` wins.
4. Component uses only existing shadcn primitives — no new dependencies.
5. At least 6 call sites adopt `<EmptyState>` in Wave 14. First wave targets:
   - `portal-requests.tsx` — "You haven't submitted any requests yet" with CTA `Submit a request`.
   - `portal-notices.tsx` (notices section) — "No notices right now."
   - `portal-finances.tsx` (ledger sub-page) — "No transactions yet."
   - `financial-rules.tsx` (Run History tab) — "No assessment runs yet."
   - `financial-rules.tsx` (Recurring tab) — "No recurring rules yet."
   - `financial-rules.tsx` (Special Assessments tab) — "No special assessments yet."
6. ≥6 tests under `tests/empty-state.client.test.tsx` cover: default render, icon slot, cta onClick fires, cta href renders link, adoption at the top 3 call sites.
7. Home alerts panel "all clear", HubAlertWidget "all clear", inbox filter empty states remain as-is (already shipped, persona-invariant, spec-aligned).

---

## Design Notes

- Wrap in a `<Card>` with `py-12 text-center`, centered icon above `text-lg font-medium` title and `text-sm text-muted-foreground` description.
- Max body width `max-w-md` so long descriptions wrap.
- If a CTA is supplied, render it as a `Button` with `variant="default"` beneath the description.
- No dark-mode-specific styling beyond what's already in shadcn tokens.

## Wave 18 — Round 2 adoption

**Date:** 2026-04-24

Wave 14 shipped the MVP across ~6 surfaces. Wave 18 broadens adoption to additional high-traffic operator and platform surfaces. The shared `<EmptyState>` component is unchanged — this is an adoption pass only.

### Surfaces adopted in Wave 18

Operator pages:
- `financial-foundation.tsx` — three call sites: Accounts list, Categories list, Account Activity rollup.
- `financial-recurring-charges.tsx` — two call sites: schedule list, run-history list.
- `financial-late-fees.tsx` — two call sites: rules list, fee-events list.
- `financial-delinquency.tsx` — two call sites: escalations list, thresholds list.
- `financial-assessments.tsx` — special-assessment list.
- `financial-payments.tsx` — payment & credit entries list.
- `meetings.tsx` — meeting list (top-level zero state).
- `announcements.tsx` — announcement list (mobile card view).
- `insurance.tsx` — policy list.
- `persons.tsx` — both "No people yet" and the filtered "No matches" states.
- `admin-users.tsx` — admin-user list.
- `community-hub.tsx` — quick-actions list and notices list.
- `executive.tsx` — executive slides (no-slides state).

Settings pages:
- `settings-billing.tsx` — "No billing account" state.

Total Wave 18 adoption: **20 call sites** across 14 files.

### Surfaces intentionally NOT migrated

- `elections.tsx` — onboarding "Get started with Elections & Votes" rich empty state ships purpose-built copy + 4-feature highlight grid + create-election CTA. Replacing it with a generic `<EmptyState>` would lose the onboarding affordance.
- `portfolio.tsx` "No active alerts — all thresholds within range" panel — semantic green check-circle aligns with Home alerts panel "all clear" pattern (5.1 spec exception).
- `vendors.tsx`, `work-orders.tsx`, `documents.tsx`, `financial-ledger.tsx` — already use `<AsyncStateBoundary>` which provides equivalent empty/loading/error markup; not double-wrapped.
- All TableRow-cell empties (e.g. inspections, maintenance-schedules per-table, financial-reports per-table) — by design these render inside the table body so the column structure is preserved.
- `inbox` per-filter empty state, `HubAlertWidget` "all clear", Home alerts panel "all clear" — spec-locked per Wave 14 (AC 7).

### Migration mechanics

Each adoption replaces an inline `<div className="...py-X text-center">…</div>` or `<Card><CardContent className="...py-X text-center">…</CardContent></Card>` block with a `<EmptyState icon={…} title={…} description={…} testId="…" />` call. No props, hooks, or callbacks were rewritten — surface-level replacement only.
