## Acceptance Criteria

Verbatim from the decision doc:

### Q1 — Hub definition
- A section hub is the top-level landing page for a zone; navigational/summary only.
- Every zone has exactly one hub page.
- Hubs are not workflow pages; contain no forms, no deep data tables.
- Hubs are reachable from the sidebar zone label directly.
- Hubs are not the same as the Home dashboard.

### Q2 — Hub composition
- (1) Zone title matching sidebar label; (2) one-sentence plain-language description; (3) sub-page link list for that zone; (4) optional ≤3 quick-action shortcuts; (5) optional ≤1 summary widget.
- No data tables, no workflow forms, no duplicate Home content.

### Q3 — Section → zone mapping
- All 13 sections mapped to one of the five customer-persona zones.
- Each zone has exactly one designated hub page.
- Mapping table committed to `decisions/1.2-section-hub-reclassification.md` (or sibling file per decision doc note).
- No section silently dropped — unmapped → open item.

### Q4 — Hub URL convention
- `/app/financials`, `/app/operations`, `/app/governance`, `/app/communications` → zone hubs.
- `/app` → Home (canonical dashboard, not a hub).
- `/portal/finances`, `/portal/requests`, `/portal/community` → portal hubs.
- `/portal` → Owner Home (not a hub).
- No hub lives at a sub-path of its zone root.

### Q5 — Hub exemption for Home
- Four hubs in `/app`: Financials, Operations, Governance, Communications.
- Three hubs in `/portal`: My Finances, My Requests, My Community.
- `/app` and `/portal` are dashboards, not hubs.

### Q6 — Role-filtered hubs
- Hub sub-page lists filtered by persona.
- No link shown for a page the persona cannot access.
- Hub filtering uses the same role-gating model as the sidebar.

### Q7 — Naming convention
- Hub page `<h1>` = zone label exactly. No "Hub" suffix. No "Overview" suffix.
- "Overview" reserved for dashboard-type pages (e.g., Operations Overview at `/app/operations/dashboard` per 0.1 AC 7).
- Document title: `Financials — YCM`, `Operations — YCM`, etc.

### Cross-Reference Notice
- `/app/operations/dashboard` (title "Operations Overview" per 0.1 AC 7) coexists with the new `/app/operations` hub (title "Operations" per Q7). 0.1 is preserved unchanged.
