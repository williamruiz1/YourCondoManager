## Acceptance Criteria

Verbatim from the decision doc:

### Q1 — Document title format
- Every page: `document.title` = `{Page Title} — YCM`.
- Separator is em dash (—), not hyphen or en dash.
- "YCM" always uppercase.

### Q2 — Hook specification
- Named `useDocumentTitle`.
- Lives at `client/src/hooks/useDocumentTitle.ts`.
- Accepts one `title: string` arg; sets `document.title` to `${title} — YCM`.
- Updates on title change.
- Is the ONLY mechanism setting `document.title` in the codebase.

### Q3 — H1 rule
- Every page has exactly one `<h1>`.
- `<h1>` content matches the `useDocumentTitle` title argument exactly.

### Q4 — Six Dashboard renames

| Current label | Canonical title | Zone |
|---|---|---|
| Dashboard (association-level) | Association Overview | Home |
| Dashboard (portfolio-level) | *see Cross-Reference Notice below* | Home |
| Dashboard (operations) | Operations Overview | Operations |
| Dashboard (financials) | Financial Overview | Financials |
| Dashboard (owner) | My Overview | Home (Portal) |
| Dashboard (board) | Board Overview | Home |

If page content does not match the proposed title, surface the discrepancy before renaming. Update breadcrumb labels to match on these six pages.

### Q5 — Dashboard prohibition (lint)
- No page uses "Dashboard" as `<h1>` or `document.title`.
- Prohibition noted in `useDocumentTitle.ts` comment.
- Lint or code review catches violations.

### Q6 — Hook-only mechanism
- No `document.title =` assignment exists outside `useDocumentTitle.ts`.
- All three live violations (`dashboard.tsx:630`, `portfolio.tsx:263`, `operations-dashboard.tsx:98`) remediated.
- `grep document.title` returns only the hook file post-implementation.

### Q7 — Portal parity
- All `/portal/*` pages use `useDocumentTitle`.
- Portal titles follow `{Page Title} — YCM` format; `<h1>` matches.
- Portal titles use first-person labels consistent with 1.1 Owner zone taxonomy (My Finances, My Requests, My Community, etc.).

### Q8 — 44-route title table
- Table covers all 44 routes.
- Columns: Route, Canonical display title, Document title, Zone.
- Committed alongside implementation changes (location per decision doc: `decisions/1.4-page-titles.md` or inline).

### Cross-Reference Notice — 0.1 Reconciliation (BINDING)
- If the 44-route table identifies `/app` as the "portfolio-level Dashboard" target → row MUST be `/app` → "Home" (0.1 AC 1), NOT "Portfolio Overview".
- If the table identifies `/app/portfolio` → row MUST be `/app/portfolio` → "Portfolio Health" (0.1 AC 5), NOT "Portfolio Overview".
- Neither page may be titled "Portfolio Overview" until 0.1 is amended.
- STOP at the table production step if any row would require amending 0.1, 0.2, or 0.3 — file `ppm_human_task_create` for founder confirmation.

## Constraints (Do NOT)

- Do NOT set `document.title` outside `useDocumentTitle.ts`.
- Do NOT use "Dashboard" as an `<h1>` or `document.title`.
- Do NOT use hyphen `-` or en dash `–` as the separator — only em dash `—`.
- Do NOT lowercase "ycm".
- Do NOT rename `/app` to "Portfolio Overview" (or anything other than "Home") — 0.1 AC 1 locks it.
- Do NOT rename `/app/portfolio` to anything other than "Portfolio Health" — 0.1 AC 5 locks it.
- Do NOT skip portal pages — Q7 mandates parity; the 3,935-line `owner-portal.tsx` mega-file gets in-place sprinkle per 4.2 Q5 (tracked so 3.5 doesn't conflict).
- Do NOT rename a page to a title that does not reflect its content — surface the discrepancy first.
