## Acceptance Criteria

Verbatim from the decision doc:

### Q1 — Six canonical zones
- Sidebar in `/app` contains up to six top-level zones: Home, Financials, Operations, Governance, Communications, Platform.
- Customer personas do not see the Platform zone (strict isolation per 0.3 Q5 + 0.2 Persona 6 matrix).
- Platform Admin sees Platform zone plus any other zone their 0.2 matrix row grants.

### Q2 — Assignment rubric
- Every page assigned to exactly one zone based on primary user intent.
- Pages serving two zones → primary-action zone.
- Genuinely ambiguous pages flagged as open items before assignment; do not silently assign.

### Q3 — Zone definitions (documented verbatim in 1.1)
- Home, Financials, Operations, Governance, Communications, Platform — as defined in the decision doc.
- `zone: Platform` implies `persona: Platform Admin` only.

### Q4 — Zone tag format
Every page file carries the following comment block at the top (below imports/license):
```
// zone: [Zone Label]
// persona: [comma-separated: Manager | Board Officer | Assisted Board | PM Assistant | Owner | Platform Admin]
```
- All 45 pages receive a tag (40 customer-tenant + 5 Platform-zone).
- Zone label matches one of the six canonical labels exactly.
- Persona list reflects the 0.2 Persona Boundary Matrix row.
- `zone: Platform` → `persona: Platform Admin` only.

### Q5 — Owner Portal zones
- `/portal` sidebar has four zones: Home, My Finances, My Requests, My Community.
- Portal files use the same tag format with `persona: Owner`.
- Portal zones do NOT mirror `/app` zone labels verbatim.

### Cross-Reference Notice — 0.1 reconciliation
- The zone-assignment table MUST use `/app` page title = "Home" per 0.1 AC 1 (NOT "Portfolio Overview").
- The zone-assignment table MUST use `/app/portfolio` page title = "Portfolio Health" per 0.1 AC 5.
- Any row with "Portfolio Overview" is invalid until 0.1 amends.
