## Acceptance Criteria

Verbatim from the decision doc:

### Q1 — Root label rule
- Association name for association-scoped pages; zone label for portfolio-scoped pages.
- No "Home," "Dashboard," or "App" as a root label.

### Q2 — Two-pattern chain
- `Association Name > Zone Label > Page Title` (association-scoped).
- `Zone Label > Page Title` (portfolio-scoped).
- All 37 pages use one of these two patterns; no other pattern remains post-implementation.

### Q3 — Hub pages
- Two-level breadcrumb `Association Name > Zone Label` (zone label is current-page indicator, not a link).
- Portfolio-scoped hubs: zone label alone, non-linked.
- No third level on hub breadcrumbs.

### Q4 — Flatten to ≤3
- No page exceeds 3 levels post-implementation.
- Flattened pages have titles specific enough to stand alone.
- List of flattened pages included in implementation output.

### Q5 — Centralized component
- One shared breadcrumb component.
- All 37 pages consume the shared component.
- No hardcoded breadcrumb markup outside the shared component.
- Route metadata drives breadcrumb content.

### Q6 — Persona invariance
- Breadcrumb renders identically for all personas on any given page.
- No persona-conditional logic in the breadcrumb component.

### Q7 — Mobile collapse
- Below 768px: single-level back affordance `< Parent` (uses `use-mobile` hook).
- Above 768px: full trail up to 3 levels.
- Mobile breadcrumb is a functional back-navigation link, not decorative text.
