## `persona-access.ts` Contract

The sidebar consumes `shared/persona-access.ts` — the manifest is the Source of Truth. Any hardcoded role array in the sidebar is a lint violation.

- **Manifest location:** `shared/persona-access.ts` (skeleton in Phase 0b; populated in Phase 9).
- **Sidebar imports:** `canAccess` predicate + route manifest. Sidebar does NOT import `AdminRole` type directly — that type is re-exported from `shared/schema.ts` through `shared/persona-access.ts` as a convenience, but the sidebar only needs the predicate and the manifest, not the raw type.
- **Contract invariant:** Any change to persona access requires updating the shared manifest — sidebar visibility and `<RouteGuard>` accessibility stay in sync automatically. Two independent hand-maintained role lists would drift (the existing Portfolio `board-admin` bug at `app-sidebar.tsx:76` is the canonical example of drift). The manifest eliminates that class of bug.
- **Lint enforcement:** Add (or verify) an ESLint rule that disallows `roles: ["..."]` literal arrays inside `client/src/components/app-sidebar.tsx`. If no such rule exists, add a narrow one in this PR — scope it to the sidebar file so other components are not affected.

## SUBSET-RENDER Model (3.1 Q5)

The core founder-binding decision. The sidebar is not "show everything, grey out what's forbidden". It is "render exactly what the persona is permitted to see, and nothing else."

**Contract:**

- **Manager:** All six zones. Portfolio Health visible. Associations visible. All sub-items visible.
- **Board Officer:** Five zones (Home, Financials, Operations, Governance, Communications). No Platform zone. No Portfolio Health. No Associations. Settings visible. All other sub-items identical to Manager.
- **Assisted Board:** Subset of zones per 0.2 PM-Managed Default Access Table + PM toggle state. No Platform zone. No Portfolio Health. No Associations. No Settings. Sub-items filtered by `usePersonaToggles` matrix.
- **PM Assistant:** Subset of zones per Manager's PM-internal toggle configuration. No Platform zone. No Portfolio Health unless Manager grants. No Settings unless Manager grants. Sub-items filtered by `usePersonaToggles` matrix.
- **Platform Admin:** All six zones (full operator view, including Platform zone).
- **Viewer:** Per 0.2 matrix; extremely narrow subset or redirected to a viewer surface entirely (verify at execution time against the Phase 9 manifest).

**Rendering rules (non-negotiable):**

1. Disallowed zones are absent from the DOM. No `hidden`, no `display: none`, no `aria-hidden`. The zone is not rendered at all.
2. Disallowed sub-items inside a permitted zone are absent from the DOM.
3. Empty zones (zero permitted sub-items) do not render their `SidebarGroup` wrapper either.
4. No greyed-out, `opacity-50`, `disabled`, `aria-disabled`, or tooltip-locked sidebar items exist for any persona.
5. The filter is data-driven: input `(persona, toggleMatrix)`, output filtered zone tree. No persona-name `switch` statements or `if` branches in JSX.

**Why this matters:** The alternative ("render everything, grey out forbidden") leaks information about platform capabilities to personas who should not know they exist — and creates the same sidebar-vs-RouteGuard drift class of bug. SUBSET-RENDER forces the sidebar to be a pure projection of the manifest.

## `usePersonaToggles` Hook (3.1 Q6)

The hook skeleton lives in Phase 0b; implementation populates in Phase 9. By the time this handoff executes (Phase 11), the hook exists. **Use it. Do not re-query `tenant_configs` directly from the sidebar.**

**Hook signature (expected):**

```ts
function usePersonaToggles(
  activeAssociationId: string | null,
  adminRole: AdminRole | null,
): {
  matrix: { [featureKey: string]: { view: boolean; write: boolean } };
  isLoading: boolean;
  isError: boolean;
};
```

**Sidebar consumption:**

- Call `usePersonaToggles(activeAssociationId, adminRole)` at the top of the sidebar component.
- While `isLoading`, render a loading skeleton — **not** the full unfiltered nav tree. Rendering full-then-filtering would flash forbidden items.
- For Assisted Board / PM Assistant, combine `matrix` with the manifest to compute each item's final visibility: `manifestAllows(persona, item) && matrix[item.feature]?.view`.
- Platform and admin surfaces are never included in the toggle matrix (0.2 PM Toggle Configuration Model rule 7) — do not expect `matrix["platform.*"]` entries.

**Query-key discipline:** The hook's query key must include `activeAssociationId`. Switching association invalidates and refetches automatically. Do not add a manual `refetch()` call.
