## Acceptance Criteria

Verbatim from the decision doc's §Acceptance Criteria (summary):

1. A shared `<BreadcrumbNav>` at `client/src/components/breadcrumb-nav.tsx` (rewritten) — the ONLY breadcrumb renderer.
2. `client/src/lib/route-metadata.ts` exports typed metadata array with 39 entries covering all breadcrumb surfaces.
3. Association-scoped pages follow Pattern A: `Association Name > Zone Label > Page Title`.
4. Portfolio-scoped pages follow Pattern B: `Zone Label > Page Title`.
5. Hub pages render two-level (association-scoped) or one-level (portfolio-scoped) with zone label as non-linked current-page indicator.
6. No breadcrumb chain exceeds 3 levels; unit test enforces this across all 39 entries.
7. No page hardcodes breadcrumb markup; all driven by route metadata.
8. Breadcrumb content is persona-invariant; no role-conditional logic in the component.
9. Below 768px: `< Parent` as a functional back link (via `use-mobile`). Above 768px: full trail ≤3.
10. Leaf crumb == `<h1>` == `document.title` page segment for every page, enforced by shared `RouteMetaEntry.title`.
11. Class A violations remediated — `dashboard.tsx:633` and `operations-dashboard.tsx:101` contain no hardcoded breadcrumb markup; no breadcrumb contains "Dashboard" (per 0.1 + 1.4 Q5).
12. Class G parallel surfaces deleted — `association-context.tsx:672` and `financial-budgets.tsx:232` contain no breadcrumb markup.
13. Generated catalog at `docs/projects/platform-overhaul/artifacts/breadcrumb-catalog.md` lists all 39 routes with resolved chains + Flattened indicator.
14. `<RouteGuard>` wraps routes above the breadcrumb in the component tree; breadcrumb has zero permission awareness.

### Implementation-shape ACs (from Q1–Q10 in the decision doc)
- `RouteMetaEntry` type fields: `path`, `zone`, `scope`, `title`, `hub`, `parent`. No `roles` / `permissions` fields.
- Shared component imports `use-active-association` internally; pages do not prop-drill association context.
- Snapshot test in `route-metadata.test.ts` captures the full metadata array.
- Generator script at `scripts/generate-breadcrumb-catalog.ts` writes the Markdown catalog.
- Mobile collapse uses `use-mobile` hook; no CSS-only media query branch.
- Hub rendering derived from `hub: true` flag, not path-matching against 1.2 hub URL list.
- Seven hub routes carry `hub: true`: `/app/financials`, `/app/operations`, `/app/governance`, `/app/communications`, `/portal/finances`, `/portal/requests`, `/portal/community`.
