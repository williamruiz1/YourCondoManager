## Objective

Ship the canonical title catalog per route, the `useDocumentTitle` hook at `client/src/hooks/useDocumentTitle.ts`, and the `{Page Title} — YCM` document-title format across all 44 audited routes (35 `/app/*` operator + 9 `/portal/*` sub-views). Collapse three competing title surfaces (`WorkspacePageHeader`, raw `<h1>`, dynamic greetings) onto a single convention. Rename six Dashboard-labeled pages per the Q4 table — respecting the 0.1 Cross-Reference Notice that forbids re-titling `/app` → "Portfolio Overview" or `/app/portfolio` → "Portfolio Overview" (0.1 locks "Home" and "Portfolio Health" respectively). ## Acceptance Criteria (verbatim / tightly paraphrased from decision doc)

1. Every page sets `document.title` in the format `{Page Title} — YCM`; separator is em dash (—), not hyphen or en dash; "YCM" is always uppercase; no page deviates (Q1)
2. Hook is named `useDocumentTitle`, lives at `client/src/hooks/useDocumentTitle.ts`, accepts one `title: string` argument, sets `document.title` to `${title} — YCM`, updates on title change; no other mechanism sets `document.title` (Q2)
3. Every page has exactly one `<h1>`; `<h1>` content matches the `useDocumentTitle` title argument exactly; no page has a `<h1>` that differs from its document title (Q3)
4. All six "Dashboard"-labeled pages are renamed to their canonical titles; renamed titles match actual page content; any mismatch is flagged before commit; breadcrumb labels updated to match (unblocking 1.3) (Q4)
5. No page uses "Dashboard" as its `<h1>` or `document.title` after implementation; prohibition noted in decision doc; comment in `useDocumentTitle.ts` notes the rule for lint/review (Q5)
6. No `document.title =` assignment exists outside `useDocumentTitle.ts`; all three live violations are remediated to use the hook; grep for `document.title` returns only the hook file (Q6)
7. All `/portal/*` pages use `useDocumentTitle`; portal document titles follow `{Page Title} — YCM` format; portal `<h1>` matches document title; portal titles use first-person labels consistent with 1.1 Owner zone taxonomy (Q7)
8. 44-route title table covers all routes with columns Route / Canonical display title / Document title / Zone; table committed to the implementation-artifacts path alongside changes; no route is missing (Q8)
9. For any row targeting `/app`, title is "Home" (per 0.1 AC 1); for any row targeting `/app/portfolio`, title is "Portfolio Health" (per 0.1 AC 5) — neither may be titled "Portfolio Overview" until 0.1 is formally amended (Cross-Reference Notice)
