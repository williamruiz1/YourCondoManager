Establish the canonical six-zone taxonomy (Home, Financials, Operations, Governance, Communications, Platform) for `/app/*` and the four-zone taxonomy (Home, My Finances, My Requests, My Community) for `/portal/*`, then apply the zone-tag comment block to every page file per Q4 tag format. ## Acceptance Criteria (verbatim / tightly paraphrased from decision doc)

1. The sidebar in `/app` contains up to six top-level zones — five for customer personas, one (Platform) for Platform Admin only; labels are Home, Financials, Operations, Governance, Communications, Platform (Q1)
2. No seventh zone exists without a decision doc amendment; customer personas (Manager, Board Officer, Assisted Board, Owner) do not see the Platform zone per 0.3 Q5 + 0.2 Persona 6 (Q1)
3. Every page is assigned to exactly one zone following the primary-intent rubric, not file-tree location; pages serving two zones are assigned to their primary-action zone; genuinely ambiguous intent is flagged as an open item before assignment (Q2)
4. Each zone definition is documented verbatim in 1.1; no page is assigned to a zone whose definition does not match its content; `zone: Platform` implies `persona: Platform Admin` only (Q3)
5. All untagged `/app/*` pages (40 customer-tenant + 5 Platform-zone surfaces) receive a zone tag comment in the `// zone: [Label]` / `// persona: [list]` format (Q4)
6. Zone label in the comment matches one of the six canonical labels exactly; persona list reflects 0.2 boundary matrix access including the Platform Admin column added at `f8dbf76` (Q4)
7. No page file is left untagged after 1.1 implementation (Q4)
8. The Owner Portal sidebar contains exactly four top-level zones with labels Home, My Finances, My Requests, My Community; portal tags use the same comment format with `persona: Owner`; portal zones do not mirror `/app` zone labels verbatim (Q5)
9. Canonical zone-assignment table uses `/app` page title == "Home" (per 0.1 AC 1) and `/app/portfolio` page title == "Portfolio Health" (per 0.1 AC 5); any row referencing "Portfolio Overview" is invalid until 0.1 is formally amended (Cross-Reference Notice)
