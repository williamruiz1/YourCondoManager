## Acceptance Criteria (verbatim / tightly paraphrased from decision doc)

1. `client/src/lib/route-metadata.ts` exists and exports the `RouteMetaEntry` type and the metadata array with exactly 39 entries; every route has exactly one entry; no breadcrumb-relevant field hardcoded in any page; importable by both breadcrumb component and `useDocumentTitle` (Q1)
2. Shared breadcrumb component imports and calls `use-active-association` internally; association-scoped pages display association name as first level without any prop/context; portfolio-scoped pages display zone label as first level; if no active association on association-scoped route, breadcrumb omits root level gracefully (Q2)
3. Breadcrumb derivation function never produces more than 3 segments; unit test iterates all entries and asserts chain length ≤3; dev-mode `console.error` fires on overflow; no intermediate breadcrumb level field in `RouteMetaEntry` (Q3)
4. Metadata array has exactly 39 entries; `scripts/generate-breadcrumb-catalog.ts` produces a Markdown table with columns Route / Scope / Zone / Breadcrumb Chain / Flattened (Y/N); `docs/projects/platform-overhaul/artifacts/breadcrumb-catalog.md` generated (not hand-maintained); snapshot test in `route-metadata.test.ts` (Q4)
5. Every page component passes `routeMetaEntry.title` to `useDocumentTitle`; shared breadcrumb component reads `routeMetaEntry.title` as leaf label; no page has breadcrumb leaf differing from `<h1>` or document title; no `usePageIdentity` wrapper hook exists — coupling via shared `RouteMetaEntry.title` field (Q5)
6. Below 768px, breadcrumb renders `< {parent label}` as single clickable back link; above 768px, full trail up to 3 levels; `use-mobile` hook is the viewport detection mechanism; mobile back link navigates to `parent` route from metadata; hub pages with `parent: null` show no breadcrumb on mobile; no separate `<BreadcrumbMobile>` component (Q6)
7. After 3.4 PR lands, `dashboard.tsx` and `operations-dashboard.tsx` contain no hardcoded breadcrumb markup; no breadcrumb in the codebase contains "Dashboard"; PR description explicitly notes Class A remediation (Q7)
8. `association-context.tsx` contains no breadcrumb markup; `financial-budgets.tsx` contains zero breadcrumb rendering calls; single `<BreadcrumbNav>` instance renders per page view hosted at layout level; visual regression test confirms correct breadcrumbs after dual-surface deletion (Q8)
9. `RouteMetaEntry` contains no `roles` / `permissions` / persona fields; breadcrumb component imports nothing from permission/role system; no persona-conditional breadcrumb logic anywhere; `<RouteGuard>` wraps routes above breadcrumb in component tree (Q9)
10. Hub pages render exactly two levels (association-scoped) or one level (portfolio-scoped); zone label on hub crumb is plain text, not `<a>`; hub pages do not show a third level; `hub` field is the sole input for hub-crumb logic; all 1.2 hub routes have `hub: true` in metadata (Q10)
