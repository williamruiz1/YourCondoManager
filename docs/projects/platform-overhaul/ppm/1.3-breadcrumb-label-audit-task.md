## Objective

Lock the breadcrumb label rules, chain patterns, hub behavior, centralization model, and mobile collapse behavior for all 37 audited pages. 1.3 is the decision skeleton that 3.4 consumes during build (3.4 implements the shared `<BreadcrumbNav>` + route-metadata map). This card covers the Layer 1 decision verification: canonical root label (association name for association-scoped, zone label for portfolio-scoped — never "Home", "Dashboard", or "App"), exactly two valid chain patterns, hub two-level pattern, ≤3-level cap with page-title disambiguation on flattening, centralized component, persona-invariant content, and `< Parent` mobile back affordance below 768px.

---

## Acceptance Criteria (verbatim / tightly paraphrased from decision doc)

1. Association-scoped breadcrumbs always show association name as level one; portfolio-scoped breadcrumbs start at zone label; no breadcrumb uses "Home", "Dashboard", or "App" as root label (Q1)
2. All 37 pages use one of the two patterns `Association > Zone > Page` (association-scoped) or `Zone > Page` (portfolio-scoped); no other breadcrumb pattern exists in the codebase after implementation (Q2)
3. Hub pages show a breadcrumb with association name (if association-scoped) and zone label; zone label on a hub breadcrumb is not a link — it is the current page indicator; hub breadcrumbs do not show a third level (Q3)
4. No page shows more than three breadcrumb levels after implementation; flattened pages have page titles specific enough to identify the page without the dropped level; implementation output includes a list of all pages that required flattening (Q4)
5. One shared breadcrumb component exists; all 37 pages use it; no page contains hardcoded breadcrumb markup outside the shared component; route metadata drives content (Q5)
6. Breadcrumb renders identically for all personas on any given page; no persona-conditional logic exists in the breadcrumb component (Q6)
7. Below 768px, breadcrumb shows one level — immediate parent with a back-chevron; above 768px, breadcrumb shows the full trail up to three levels; the mobile breadcrumb is a functional back-navigation link, not decorative text (Q7)
