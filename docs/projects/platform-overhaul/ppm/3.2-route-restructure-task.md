## Objective

Ship the canonical post-3.2 `/app/*` route tree (61 entries): add four new hub pages (`/app/financials`, `/app/operations`, `/app/governance`, `/app/communications`), relocate current Governance and Communications content to `/overview` sub-paths, add five plural-to-singular Financials legacy redirects, retire the `/app/admin` alias as a hard 404, archive all 43 existing legacy redirects + 26 PublicRouter catch-alls for rollback safety, and defer `/portal/*` restructure to separate work. Preserve 0.1-locked URLs (`/app`, `/app/portfolio`, `/app/operations/dashboard`) unchanged.

1. `/app/financials` renders a hub page with `<h1>` "Financials" per 1.2 Q2 composition; `/app/financials/{sub-page}` redirects to `/app/financial/{sub-page}` with no 404; existing `/app/financial/*` URLs resolve unchanged; nine finance legacy redirects continue to function (Q1)
2. `/app/operations` renders a hub page with `<h1>` "Operations" per 1.2 Q7 and links to Operations Overview; `/app/operations/dashboard` unchanged per 0.1 AC 7; hub and dashboard are distinct routes serving distinct components (Q2)
3. `/app/governance` renders a hub page with `<h1>` "Governance" per 1.2 Q7 linking to `/app/governance/overview`; `/app/governance/overview` renders the current `GovernancePage` content — no content loss; same pattern applies to `/app/communications` ↔ `/app/communications/overview`; four governance legacy redirects still resolve (Q3)
4. All 43 existing `RouteRedirect` entries (9 finance, 4 governance, 2 person/occupancy, 2 single-association role-based, 26 PublicRouter catch-alls) remain and continue to function; no `RouteRedirect` is deleted or disabled in 3.2 (Q4)
5. No `/portal/*` route added, removed, renamed, or redirected; 1.2 Q4 portal hub URLs (`/portal/finances`, `/portal/requests`, `/portal/community`) not implemented in this pass (Q5)
6. The route declaration mapping `/app/admin` to `RoadmapPage` is removed; `/app/admin` returns NotFound (404); `/app/admin/roadmap`, `/app/admin/users`, `/app/admin/executive` continue to function with `<RouteGuard roles={["platform-admin"]}>` (Q6)
7. All 26 `PublicRouter` `/app/*` catch-all `RouteRedirect` entries at `App.tsx:365-461` remain and continue to function; no catch-all is consolidated or deleted in 3.2 (Q7)
