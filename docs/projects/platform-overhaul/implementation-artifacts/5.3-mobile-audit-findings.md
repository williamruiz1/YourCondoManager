## Fixes Applied

### Fix 1 — Central inbox tab list scrolls horizontally
**File:** `client/src/pages/communications-inbox.tsx`
**Change:** wrapped `<TabsList>` in `-mx-1 overflow-x-auto px-1` with `w-max` on the list.
**Why:** Per `mobile-ui-rules.md` → "Use horizontally scrollable pill tabs for primary or secondary mobile navigation." Previously could wrap awkwardly at 375px.

### Fix 2 — Assessment rules tab list scrolls horizontally
**File:** `client/src/pages/financial-rules.tsx`
**Change:** identical pattern to Fix 1.
**Why:** "Special Assessments" label is long; 3 tabs would push past the viewport.

### Fix 3 — Portal ledger table overflow-x-auto
**File:** `client/src/pages/portal/portal-finances.tsx`
**Change:** added `overflow-x-auto` to the ledger `<CardContent>` wrapping the table, and wrapped the zero-rows state in the new `<EmptyState>`.
**Why:** 4-column table with long descriptions couldn't fit at 375px — the card itself was growing wider than the viewport rather than scrolling internally.

### Surprising patterns spotted during audit

1. **`isMobile ? "grid-cols-1" : "grid-cols-2"` is a recurring anti-pattern** across the codebase — 12+ instances inside dialog forms alone. It works (the `useIsMobile()` hook reads `window.matchMedia`) but adds a re-render dependency and reads as desktop-first. Wave 28 normalized the dialog instances; non-dialog instances (e.g. summary card grids on `financial-ledger.tsx`, `persons.tsx` directory, `board.tsx`) still carry the pattern and could be migrated in a future polish pass.
2. **Default shadcn `<DialogContent>` ships `w-full max-w-lg`** — at 320px viewport with `p-6` (24px each side), content has only ~272px to work in. The Wave 28 viewport-clamp pattern (`max-w-[calc(100vw-2rem)]`) is the correct primitive but should arguably be backed into `DialogContent` itself. Out of scope for Wave 28 (no shared component changes); filed mentally for a future "shadcn upgrade" pass.
3. **Documents.tsx has a sidebar-style two-column dialog layout** (`md:grid-cols-[1.1fr_0.9fr]`) where the right column is a workflow checklist + summary card. That pattern was already correctly responsive (stacks below md). It's a useful template for future complex-form dialogs.
