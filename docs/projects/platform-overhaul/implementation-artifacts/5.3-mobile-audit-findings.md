## Fixes Applied

### Fix 1 — Central inbox tab list scrolls horizontally
**File:** `client/src/pages/communications-inbox.tsx`
**Change:** wrapped `<TabsList>` in `-mx-1 overflow-x-auto px-1` with `w-max` on the list.
**Why:** Per `mobile-ui-rules.md` → "Use horizontally scrollable pill tabs for primary or secondary mobile navigation." Previously could wrap awkwardly at 375px.

### Fix 2 — Assessment rules tab list scrolls horizontally
**File:** `client/src/pages/financial-rules.tsx`
**Change:** identical pattern to Fix 1.
**Why:** "Special Assessments" label is long; 3 tabs would push past the viewport.

### Fix 3 — Portal ledger table overflow-x-auto
**File:** `client/src/pages/portal/portal-finances.tsx`
**Change:** added `overflow-x-auto` to the ledger `<CardContent>` wrapping the table, and wrapped the zero-rows state in the new `<EmptyState>`.
**Why:** 4-column table with long descriptions couldn't fit at 375px — the card itself was growing wider than the viewport rather than scrolling internally.

## Refactors (not "fixes", but worth noting)

All four zone hub placeholder pages (`client/src/pages/hubs/*-hub.tsx`) now render `<EmptyState>` instead of ad-hoc Card + Link markup. The visual output is nearly identical at `md+` breakpoints; at `sm:` and below the new component has consistent padding (`py-12 px-4 text-center`) that matches the rest of the adoption.
