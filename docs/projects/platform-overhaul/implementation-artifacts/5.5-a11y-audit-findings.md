# 5.5 — Accessibility Audit Findings (Wave 21)

> Spec: `decisions/5.5-accessibility-audit.md`
> Wave: 21 (2026-04-25)

Per-surface findings + the mechanical fix applied. The audit followed the four-criteria triage: (a) `aria-*` on interactive controls, (b) keyboard nav + focus-visible outlines, (c) color contrast (spot-check), (d) semantic HTML.

Severity glossary:
- **HIGH** — keyboard or screen-reader user cannot complete the task without it.
- **MED** — task is completable but the experience is degraded (icon button announces only "button", no skip-link, etc.).
- **LOW** — polish; no functional impact (decorative icon double-announcing).

---

## Shells (cross-cutting)

### Operator shell — `client/src/App.tsx::MainContent`

| Finding | Severity | Fix |
|---|---|---|
| No skip-link present. Keyboard users tab through the entire sidebar before reaching page content. | HIGH | Added `<a href="#main-content">` styled `sr-only focus:not-sr-only` directly above `<main>`. |
| `<main>` had no `id` and was not programmatically focusable (`tabIndex={-1}`). | MED | Added `id="main-content"` + `tabIndex={-1}` so the skip-link target receives focus on activation. |

### Portal shell — `client/src/pages/portal/portal-shell.tsx::PortalShell`

| Finding | Severity | Fix |

| `<main>` had no `id` and no `tabIndex`. | MED | Added `id="portal-main-content"` + `tabIndex={-1}`. |
| `<nav aria-label="Portal navigation">` and `<nav aria-label="Breadcrumb">` already present — no fix needed. | — | (Verified.) |

## 1. Operator Home — `dashboard.tsx`

| Finding | Severity | Fix |
|---|---|---|
| `<Button asChild>` containing `<Link>Review</Link>` has no programmatic association with the alert it reviews. Screen readers announce just "Review". | MED | Added `aria-label={`${t("common.review")} ${label}`}` to the inner `<Link>` so each button announces "Review urgent work orders" etc. |
| Decorative Lucide icons inside `AlertRow`, `QuickActions`, `Vote`, `Clock`, `Timer`, `ClipboardCheck` were rendered without `aria-hidden`. | LOW | Added `aria-hidden="true"` on every Lucide icon adjacent to user-facing text. |
| `<div className="rounded-xl border bg-muted/10 ...">` containing the alerts panel had no semantic role. | MED | Promoted to `<section aria-labelledby="home-alerts-heading">` with the inner heading `id="home-alerts-heading"`. |
| Empty / loading / "all clear" states had no `role`. | MED | Added `role="status"` on the loading skeleton wrapper and the all-clear container. |
| Quick-action buttons used icon + label but no `aria-label` for the button as a whole. Disabled variants render a `<span>` (no programmatic action). | LOW | Added `aria-label={action.label}` to the button so the announced text is consistent regardless of disabled state. |
| Election rows used `<Link>` with bare "Review" / "Certify" text — same orphan-link issue. | MED | Added `aria-label={`${t("common.review")} ${election.title}`}` and `aria-label={`Certify ${election.title}`}`. |
| Association list rows had buttons "In Context" / "Use Context" with no association name. | MED | Added `aria-label={`${state} ${association.name}`}`. |
| Color contrast on the green "All clear" headline against the muted card background — already meets AA (`text-green-700` against `bg-background` ≥ 4.5:1). | — | Verified, no fix. |

## 7. Assessment Rules — `financial-rules.tsx`

| Finding | Severity | Fix |
|---|---|---|
| Filter `<Select>` triggers had no associated `<label htmlFor>` and no `aria-label` — screen readers announce only "combobox" with the current value. | HIGH | Added `<label htmlFor="…">` for each filter, plus `aria-label` on the matching `<SelectTrigger id="…">`. |
| `Refresh` button rendered as bare text "Refresh" with a small viewport — fine, but added `aria-label` for consistency with the rest of the audit. | LOW | Added `aria-label={t("common.refresh")}`. |
| Run History `<Table>` had no `<caption>` / `aria-label`. Screen-reader users entering table mode could not identify the data set. | HIGH | Added `aria-label={t("financialRules.runHistory.title")}` + a hidden `<caption className="sr-only">`. |
| Decorative icons in the status badges (`CheckCircle2`, `XCircle`, `RotateCcw`, `Clock`) had no `aria-hidden`. | LOW | Added `aria-hidden="true"`. |
| Filter section card icon (`Filter`) and run-history card icon (`Calendar`) had no `aria-hidden`. | LOW | Added `aria-hidden="true"`. |
| Loading skeleton wrapper / "Select an association" message had no role. | MED | Added `role="status"`. |
| Empty / error states use shared `<EmptyState>` / `<ErrorState>` — inherit Wave 21 role updates. | — | (Inherited fix.) |
| Custom-window `<Input type="date">` controls had only a sibling label, not an `aria-label`. | MED | Added explicit `aria-label` to each input. |

## 9. Owner Finances — `portal/portal-finances.tsx`

| Finding | Severity | Fix |
|---|---|---|
| `<Input type="number">` for the payment amount had only a sibling heading, not an `aria-label` or `aria-labelledby`. | HIGH | Added `id` to the heading + `aria-labelledby` on the input + a redundant `aria-label` for resilience. |
| "Set default" / "Remove" buttons announced as bare verbs across multiple payment-method cards. | MED | Added `aria-label` referencing bank + last-4 on each. |
| Payment-method link cards (`Payment methods & autopay`, `Full ledger history`) had no focus ring. | MED | Added `focus-visible:outline …` utilities. |
| `<Table>` blocks (Recent Ledger, Full Ledger) had no `<caption>` / `aria-label`. | HIGH | Added `aria-labelledby` on Recent Ledger pointing to its heading, and `aria-label` + `<caption className="sr-only">` on the full Ledger table. |
| The virtualized ledger renders bare `<div role="row">` headers without `role="columnheader"`. | MED | Added `role="columnheader"` to the four header cells and `role="table"` + `aria-label` on the wrapper. |
| Type-filter buttons (`charge`, `payment`, etc.) are a button group with no `role` and no `aria-pressed` state. | MED | Wrapped the group in `role="group" aria-label="Type"` and added `aria-pressed` to each toggle button. |
| Decorative arrow icons in quick-link cards had no `aria-hidden`. | LOW | Added `aria-hidden="true"`. |
| Empty `CardContent` blocks (`No ledger entries yet`, `No saved payment methods yet`, `Not enrolled in autopay`) had no role. | MED | Added `role="status"` to each. |
| Section headings for upcoming assessments / recent ledger / saved methods / autopay enrollments had no `id` linking them to their section. | LOW | Added `id` + `aria-labelledby` pairs. |
