# 5.5 — Accessibility Audit Findings (Wave 21)

> Spec: `decisions/5.5-accessibility-audit.md`
> Wave: 21 (2026-04-25)

Per-surface findings + the mechanical fix applied. The audit followed the four-criteria triage: (a) `aria-*` on interactive controls, (b) keyboard nav + focus-visible outlines, (c) color contrast (spot-check), (d) semantic HTML.

Severity glossary:
- **HIGH** — keyboard or screen-reader user cannot complete the task without it.
- **MED** — task is completable but the experience is degraded (icon button announces only "button", no skip-link, etc.).
- **LOW** — polish; no functional impact (decorative icon double-announcing).
